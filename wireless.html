Wireless Networks

Access Points, Wireless LAN Controllers, WiFi standards, roaming, SSIDs, and enterprise wireless architecture across Cisco, Palo Alto, and Fortinet.

Each generation of WiFi improves speed, capacity, and efficiency. Enterprise environments typically require at least WiFi 6 (802.11ax) on new deployments.

WiFi 6E adds the 6 GHz band — a clean, uncongested spectrum with no legacy devices. Ideal for high-density environments (offices, conference rooms). WiFi 6E APs are now the standard recommendation on new deployments.

Enterprise Wireless Architecture

Centralized CAPWAP architecture: all APs tunnel traffic to the WLC for policy enforcement

Centralized (CAPWAP)

APs tunnel all client traffic back to the WLC (Wireless LAN Controller) using CAPWAP (Control and Provisioning of Wireless Access Points). The WLC enforces all security policies, applies QoS, and manages roaming. Best for environments where all APs are near the WLC. Used by Cisco WLC, Fortinet FortiWLC.

FlexConnect / Local Mode

In FlexConnect (Cisco), APs can switch traffic locally at the branch even if the WAN link to the WLC goes down. Only management traffic tunnels to the WLC. Critical for branch offices — clients stay connected during WAN outages.

Cloud-Managed

Modern APs (Cisco Catalyst Center, Fortinet FortiCloud, Palo Alto Prisma Access) are managed from the cloud. No on-premises WLC hardware needed. Configuration, firmware updates, and monitoring happen through a cloud portal. Ideal for distributed multi-site deployments.

PoE — Powering APs

Enterprise APs get both data and power from the Ethernet cable — PoE (802.3af) provides 15W, PoE+ (802.3at) provides 30W, and PoE++ (802.3bt) provides 60–90W. WiFi 6E APs typically require PoE+ or PoE++. The switch must have a PoE budget large enough to power all connected APs.

SSIDs, VLANs & Security

SSID to VLAN Mapping

Each SSID (WiFi network name) maps to a VLAN. This is how an AP can broadcast both "Corp-WiFi" and "Guest-WiFi" simultaneously — each on a different VLAN, keeping traffic separated.

Corp-WiFi  → VLAN 10 (192.168.10.0/24)
IoT-Devices → VLAN 40 (10.10.40.0/24)
Guest-WiFi  → VLAN 20 (192.168.20.0/24)
                ↓
Each tunneled to WLC or bridged locally

Authentication Methods

  • WPA3-Personal: Pre-shared key, SAE handshake (replaces PSK)
  • WPA3-Enterprise: 802.1X + RADIUS — each user authenticates individually
  • 802.1X / EAP: Corporate standard — authenticates via Active Directory/LDAP
  • Captive Portal: For guest WiFi — redirect to a web login page
  • MAB (MAC Auth Bypass): For devices that can't do 802.1X (IoT)

Roaming

Layer 2 Roaming

Client moves between APs in the same VLAN/subnet. The WLC updates the MAC address table. The client keeps its IP address — seamless for voice and video. Fast handoff using 802.11r (Fast BSS Transition).

Layer 3 Roaming

Client moves between APs in different VLANs/subnets. The WLC creates a mobility tunnel to anchor the session — the client's original IP is preserved. More complex but allows roaming across buildings with different subnets.

802.11r — Fast Roaming

Fast BSS Transition pre-authenticates the client at neighboring APs before it moves. Dramatically reduces roaming handoff time from ~100ms to ~20ms — critical for voice calls and video that break above 50ms disruption.

802.11k / 802.11v

802.11k lets APs report neighbor lists to clients — clients know which AP to roam to next. 802.11v lets the network steer clients to less-congested APs (BSS Transition Management). Together they enable Band Steering (pushing 5 GHz-capable clients off 2.4 GHz).

Prisma Access — Wireless Security

When employees connect to WiFi (corporate or otherwise), Prisma Access creates a secure tunnel from their device to a cloud-based security stack. All traffic is inspected by Palo Alto's App-ID, URL filtering, and threat prevention — regardless of which physical WiFi network they're on.

🔍

IoT Security for Wireless Devices

PA firewalls use machine learning to identify wireless IoT devices (cameras, sensors, printers) that connect to the network — even if they can't authenticate via 802.1X. Profiles are applied automatically to control what those devices can access.

DHCP Option 43 — WLC Discovery

Option 43 is how a Cisco AP finds its Wireless LAN Controller automatically when it powers on — with no manual configuration on the AP itself.

🤔 What is DHCP Option 43?

When a Cisco AP boots up for the first time, it has no configuration at all — it doesn't know the IP address of its Wireless LAN Controller (WLC). It needs to find the WLC before it can do anything. DHCP Option 43 is the mechanism that solves this problem.

Option 43 is a vendor-specific field inside the DHCP reply that your DHCP server sends to the AP. The AP reads this field, extracts the WLC IP address from it, and then contacts the WLC to register. The AP is fully operational in under a minute — no one ever has to touch it individually.

🔑 Why Can't You Just Type an IP?

Cisco APs are designed for zero-touch deployment at scale. In an enterprise with 500 APs across 20 floors, you cannot physically configure each AP one by one. Instead:

  • The AP arrives from the warehouse, is mounted on the ceiling, and is plugged into a PoE switch.
  • It powers on, sends a DHCP request, and receives its IP plus the WLC address via Option 43.
  • It contacts the WLC, downloads its config, and starts broadcasting WiFi — all within 60 seconds.

This is how large enterprise WiFi deployments are actually done.

🔍 AP Discovery Order — Option 43 Is Not Alone

Cisco APs actually try multiple methods to find the WLC. Option 43 is the most reliable, but the AP will also try:

  • 1. DHCP Option 43 — the recommended method (covered on this page)
  • 2. DNS lookup — resolves CISCO-CAPWAP-CONTROLLER.domain
  • 3. Broadcast — local subnet broadcast (only works if WLC is on the same subnet)
  • 4. Previously joined WLC — remembers the last WLC from flash memory
  • 5. Catalyst Center PnP — if Catalyst Center is configured

Option 43 is used first if present, which is why it is the standard enterprise method.

📡 What Happens After Option 43?

Once the AP has the WLC IP from Option 43, it:

  • Opens a CAPWAP tunnel (UDP port 5246/5247) to the WLC
  • Authenticates via X.509 certificate (mutual authentication)
  • Downloads its software image from the WLC (if versions don't match, the AP auto-upgrades)
  • Receives its full configuration: SSIDs, VLANs, radio settings, QoS policies
  • Starts broadcasting WiFi — fully operational

The WLC remains the AP's "brain" — the AP forwards all client traffic back through the CAPWAP tunnel.

🔬 How Option 43 Is Encoded — Cisco Format

You cannot just put the IP address as plain text into Option 43. It must be encoded as a specific sequence of hexadecimal bytes. The Cisco format for Option 43 uses a sub-option structure inside the DHCP option field. Here is exactly how it works:

The sub-type byte F1 is Cisco's vendor-specific code that tells the AP "this Option 43 payload is a list of WLC IP addresses." The length byte is simply the total number of IP octets that follow (number of WLC IPs × 4). Then each IP address follows, one octet per byte, in decimal-to-hex conversion. For two WLCs, both sets of 4 bytes are concatenated after the single length byte.

4
WLC replies with CAPWAP Discovery Response → DTLS handshake
WLC sends Discovery Response → AP and WLC perform DTLS certificate exchange → encrypted CAPWAP control tunnel established
