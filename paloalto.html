🔶 Vendor Guide

Palo Alto Networks

PA-Series NGFWs from desktop to hyperscale data center, Panorama management, and Prisma cloud security.

ℹ️

PAN-OS essentials: All devices run PAN-OS. Key concepts: Zones (group interfaces — untrust, trust, DMZ), Security Policy (zone-to-zone rules using App-ID), Commit (changes sit in candidate config until you commit — nothing takes effect until then). CLI entry: ssh admin@x.x.x.x. GUI: https://mgmt-ip. Certification: PCNSA (associate) → PCNSE (engineer).

Entry Level — Desktop NGFWs

NGFW
PA-220
Desktop NGFW — small branch, retail, home office

The PA-220 is the entry-level Palo Alto NGFW. It is a small desktop unit typically mounted on a wall or placed on a shelf. Despite its size it runs the same full PAN-OS as the PA-7000 — all App-ID, URL filtering, WildFire sandboxing, and GlobalProtect features are available. E1/1 is typically the untrust (internet) zone; E1/2–E1/8 are assigned to trust (LAN) or DMZ zones. The out-of-band management port (MGT) is on a separate network for secure access.

PA-400 Series — 1U rack desktop NGFWs for mid-size branches

The PA-400 series replaces the older PA-220 for larger branch offices. The PA-415 is the entry model; the PA-450 is the top of the series with hardware SSL decryption acceleration and dedicated HA ports. All models fit in a rack with optional rack-mount kit. The PA-450 adds two dedicated HA ports (HA1 + HA2) for active/passive or active/active clustering without consuming data ports.

Mid-Range — PA-3200 Series

NGFW — 1U RACK
PA-3220 / PA-3250 / PA-3260
Enterprise perimeter and data center edge NGFWs

The PA-3200 series is for enterprise perimeters, internet edge, and data center edge deployments. Each model uses Palo Alto's custom ASICs — separate processors handle firewall sessions, threat prevention, and SSL decryption independently, so enabling all features simultaneously does not degrade throughput the way it does on competitors. The PA-3260 also supports VSYS (virtual systems) for hosting multiple firewall contexts on one physical device.

High-End — PA-5200 Series

NGFW — HIGH END
PA-5220 / PA-5250 / PA-5260
Large enterprise and service provider NGFWs — 20–72 Gbps

The PA-5200 series is for large enterprise internet edges, large campus perimeters, and service providers. The PA-5260 at the top of the range delivers 72 Gbps App-ID throughput. All models have NPC (Network Processing Cards) — modular interface cards that add interface density without replacing the chassis. Supports up to 225 VSYS for service provider multi-tenancy. Panorama is essentially required at this scale.

Data Center — PA-7000 Series

NGFW — CHASSIS
PA-7050 / PA-7080
Modular chassis NGFWs — hyperscale data center and service provider

The PA-7000 is a fully modular chassis firewall. The chassis holds Line Cards (LPC) for interfaces and Switch Management Cards (SMC) for the control plane. The PA-7050 has 5 slots; the PA-7080 has 8 slots. Each slot can hold a different line card — mix and match 100G, 40G, and 10G cards as needed. Used by large enterprises with 100G data center interconnects, major internet service providers, and large cloud providers. Panorama is mandatory — managing a PA-7000 via direct GUI is not practical at scale.

Panorama — Centralised Management

MANAGEMENT
Panorama M-200 / M-500 / M-700
Central management server for all PA firewalls

Panorama is the single pane of glass for all Palo Alto firewalls. You define Device Groups (e.g. "All branches", "Data center edge") and push policy packages to hundreds of firewalls simultaneously. Templates handle shared network config (zones, IP addresses, routing). The M-700 is the latest and most powerful, supporting 96 TB log storage. Also available as a VM (Panorama-VM on ESXi, KVM, AWS, Azure, GCP).

Cloud Security — Prisma

☁️

Palo Alto does not make APs or physical switches. Their wireless and LAN strategy is cloud-based. Prisma Access secures users on any WiFi or internet connection by routing traffic through Palo Alto cloud PoPs for inspection — regardless of which physical AP they're connected to. Prisma SD-WAN (formerly CloudGenix) provides SD-WAN with Palo Alto security integrated. For AP hardware at PA customers, Cisco or Fortinet APs are typically deployed alongside the PA firewalls.

🌐

Prisma Access

Cloud-delivered SASE platform. Every user — whether in the office, home, or travelling — tunnels through Prisma Access for internet security (ZIA equivalent) and private app access (ZPA equivalent). Managed from Panorama or Prisma Cloud portal. 150+ PoP locations globally.

🔀

Prisma SD-WAN

Intelligent WAN with integrated Palo Alto security. Branch appliances (ION series) connect to cloud PoPs for centrally-managed WAN optimisation and security — an alternative to deploying separate SD-WAN + firewall hardware at each branch.
