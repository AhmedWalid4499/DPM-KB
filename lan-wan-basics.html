LAN & WAN Basics

Understand how networks are structured, how data travels, and what each device's role is — from a single office to a global enterprise.

What is a Network?

🏢

LAN — Local Area Network

A LAN covers a single building, floor, or campus. All devices are under your control, connected by switches and access points. Speeds are fast — 1 Gbps to 100 Gbps. Examples: an office floor, a data center, a university campus.

🌍

WAN — Wide Area Network

A WAN connects multiple LANs across cities, countries, or continents. It uses leased lines, MPLS circuits, or the internet. Orange Business manages WAN services — this is a core part of our business. The internet itself is the world's largest WAN.

🏙️

MAN — Metropolitan Area Network

A MAN spans a city or metropolitan area — larger than a LAN but smaller than a WAN. Often used to connect multiple offices in the same city over fibre. Sometimes called a campus WAN.

☁️

Cloud / SD-WAN

Modern enterprises extend their networks into cloud providers (AWS, Azure, GCP). SD-WAN intelligently routes traffic across multiple WAN links. Orange Business provides managed SD-WAN services to enterprise customers.

Key Network Devices

🔀

Router

Connects different networks and routes traffic between them using IP addresses. Every packet going from your LAN to the internet passes through a router. Operates at Layer 3. Runs routing protocols like OSPF and BGP. See IP Routing →

🔌

Switch

Connects devices within the same LAN using MAC addresses. Learns which device is on which port and forwards traffic only where needed. Operates at Layer 2. Supports VLANs to logically segment traffic. See Switching →

🛡️

Firewall

Guards the network perimeter, permitting or blocking traffic based on security policy. Modern Next-Generation Firewalls (NGFW) inspect application-layer traffic, not just ports and IPs. See Firewalls →

📡

Access Point (AP)

Broadcasts a WiFi signal so wireless devices can join the network. In enterprise environments, APs are centrally managed by a Wireless LAN Controller (WLC) or cloud system. See Wireless →

The OSI Model — Simplified

The OSI model describes how data travels through a network in 7 layers. You'll hear these constantly in networking conversations.

7ApplicationHTTP, DNS, FTP, SMTP — what the user's software sees. The "why".
6PresentationEncryption, compression, data format conversion (e.g. TLS/SSL lives here).
5SessionManages and maintains connections between applications (sessions, tokens).
4TransportTCP (reliable) and UDP (fast). Port numbers live here — e.g. port 443 = HTTPS.
3NetworkIP addresses, routing decisions. Routers and firewalls operate here.
2Data LinkMAC addresses, Ethernet frames. Switches operate here. VLANs too.
1PhysicalCables, fibre, electrical signals, radio waves. The actual medium.
💡

Memory trick: "All People Seem To Need Data Processing" — Application, Presentation, Session, Transport, Network, Data Link, Physical.
