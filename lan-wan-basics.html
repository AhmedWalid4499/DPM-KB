LAN & WAN Basics

Understand how networks are structured, how data travels, and what each device's role is — from a single office to a global enterprise.

What is a Network?

🏢

LAN — Local Area Network

A LAN covers a single building, floor, or campus. All devices are under your control, connected by switches and access points. Speeds are fast — 1 Gbps to 100 Gbps. Examples: an office floor, a data center, a university campus.

🌍

WAN — Wide Area Network

A WAN connects multiple LANs across cities, countries, or continents. It uses leased lines, MPLS circuits, or the internet. Orange Business manages WAN services — this is a core part of our business. The internet itself is the world's largest WAN.

🏙️

MAN — Metropolitan Area Network

A MAN spans a city or metropolitan area — larger than a LAN but smaller than a WAN. Often used to connect multiple offices in the same city over fibre. Sometimes called a campus WAN.

☁️

Cloud / SD-WAN

Modern enterprises extend their networks into cloud providers (AWS, Azure, GCP). SD-WAN intelligently routes traffic across multiple WAN links. Orange Business provides managed SD-WAN services to enterprise customers.

Key Network Devices

🔀

Router

Connects different networks and routes traffic between them using IP addresses. Every packet going from your LAN to the internet passes through a router. Operates at Layer 3. Runs routing protocols like OSPF and BGP. See IP Routing →

🔌

Switch

Connects devices within the same LAN using MAC addresses. Learns which device is on which port and forwards traffic only where needed. Operates at Layer 2. Supports VLANs to logically segment traffic. See Switching →

🛡️

Firewall

Guards the network perimeter, permitting or blocking traffic based on security policy. Modern Next-Generation Firewalls (NGFW) inspect application-layer traffic, not just ports and IPs. See Firewalls →

📡

Access Point (AP)

Broadcasts a WiFi signal so wireless devices can join the network. In enterprise environments, APs are centrally managed by a Wireless LAN Controller (WLC) or cloud system. See Wireless →

Layer 7
Application
This is the layer you interact with directly. When you open Chrome and go to a website, your browser is a Layer 7 application. When your email client sends a message, it uses Layer 7 protocols. This layer defines what the data means — not how it travels.
Protocols at this layer
HTTP/HTTPS DNS FTP SMTP SSH SNMP
Where you see this
🌐 Web browsers 📧 Email clients 🔥 NGFW App-ID
Data at this layer is called a "Message"

🔐
Layer 6
Presentation
This layer is the translator and encryptor. It converts data into a format the application can understand, handles compression, and manages encryption/decryption. When you visit an HTTPS site, TLS encryption happens at Layer 6 — your data is scrambled before it leaves Layer 5 and unscrambled when it arrives.
Protocols & formats
TLS 1.3 SSL JPEG / PNG GZIP ASCII / UTF-8
Where you see this
🔒 HTTPS handshake 🛡️ SSL Inspection (NGFW)
Data at this layer is "encrypted & formatted"
🔒 DATA
🤝
Layer 5
Session
The Session layer opens, manages, and closes conversations between two applications. Think of it as the "phone call manager" — it dials the call, keeps it alive while you're talking, and hangs up when you're done. When you log into a website and your session stays open while you browse multiple pages, that's Layer 5 at work.
Examples
NetBIOS RPC Session cookies Auth tokens
Where you see this
🌍 Web login sessions 📞 VoIP call setup
🚚
Layer 4
Transport
Transport handles end-to-end delivery between two applications. It breaks large data into segments, numbers them, and reassembles at the destination. Two options: TCP (confirms every segment arrived — reliable, used for web/email) or UDP (fires and forgets — fast, used for video calls and gaming). Port numbers live here.
Protocols & port examples
TCP UDP Port 443 = HTTPS Port 80 = HTTP Port 22 = SSH Port 53 = DNS
Devices at this layer
🛡️ Firewalls (port filtering) ⚖️ Load balancers
Data unit = Segment (TCP) or Datagram (UDP)
TCP HDR
DATA

Layer 3
Network
The Network layer handles logical addressing and routing — getting packets from one network to another across the globe. Every device has an IP address at this layer. Routers read Layer 3 headers to decide: "which direction does this packet need to go next?" This is the layer that makes the internet possible.
Protocols at this layer
IPv4 IPv6 OSPF BGP ICMP (ping) NAT
Devices at this layer
🔀 Routers 🛡️ Firewalls 🔌 Layer 3 Switches
Data unit = Packet
IP HDR
TCP
DATA

Layer 2
Data Link
Data Link handles node-to-node delivery within the same network. It uses MAC addresses — a unique hardware ID burned into every network card. Switches read MAC addresses to forward frames to the right port. This layer also detects transmission errors and manages VLANs.
Protocols at this layer
Ethernet (802.3) WiFi (802.11) VLANs (802.1Q) ARP STP
Devices at this layer
🔌 Switches 📡 Access Points 🌉 Bridges
Data unit = Frame

Layer 1
Physical
The Physical layer is the actual hardware and transmission medium. It converts binary data (1s and 0s) into electrical signals, light pulses, or radio waves, and vice versa. There are no addresses or protocols here — just raw signals. If a cable is unplugged or a fibre is broken, the problem is at Layer 1.
Examples at this layer
Cat6 / Cat6A cable Fibre optic Radio (WiFi) SFP transceivers RJ45 connectors
Devices at this layer
🔌 Network cables 💡 Fibre transceivers 📶 Antennas 🔁 Hubs / Repeaters
Data unit = Bits (1s and 0s)
