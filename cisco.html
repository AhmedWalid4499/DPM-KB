Cisco Devices

Switches, routers, firewalls, access points, wireless controllers, and management platforms — Catalyst Center and Meraki. The world's most widely deployed enterprise networking vendor.

ℹ️

Cisco IOS / IOS-XE: The CLI you'll use most. Key commands: show ip route · show interfaces · show mac address-table · configure terminal. Cisco certifications: CCNA → CCNP → CCIE. Product families: Catalyst (campus LAN), Nexus (data center), ISR/ASR (WAN routers), ASA/FTD (firewalls). Management platforms: Catalyst Center (formerly DNA Center / DNAC) for enterprise automation, Meraki for cloud-managed branch networking.

Management & Software Platforms

Cisco's hardware is only half the story. These two software platforms are how large organisations actually manage, automate, and monitor their Cisco infrastructure — and you'll encounter both of them regularly at Orange Business.

💡

Quick distinction: Catalyst Center (DNA Center) manages traditional enterprise hardware you already own — switches, routers, WLCs, APs running IOS-XE. Meraki is a completely separate cloud-only product line where the hardware and software are sold together and managed entirely from a browser. They are not interchangeable.

🧠

Cisco Catalyst Center

Formerly called DNA Center (DNAC) — renamed in 2023

Catalyst Center is Cisco's on-premises network management, automation, and assurance platform. It gives you a single dashboard to manage every Cisco switch, router, wireless controller, and AP across the entire enterprise. Think of it as the "brain" that sits on top of all the hardware we've already covered on this page.

It runs as a software appliance (on dedicated Cisco UCS servers or a virtual appliance) and communicates with devices using NETCONF, RESTCONF, and traditional SNMP/SSH. Devices don't need to be replaced — Catalyst Center manages the same Catalyst 9300 switches and 9800 WLCs already in your network.

📌

You may still hear "DNAC" from engineers and customers — it was called DNA Center for years before the rebrand. When someone says "DNAC", they mean the same platform.

What Catalyst Center does:

  • Network Discovery — automatically discovers all Cisco devices via CDP/LLDP and builds a topology map
  • Automation — deploy configs, VLANs, QoS policies, and software upgrades across hundreds of devices simultaneously
  • SD-Access (SDA) — Cisco's campus fabric overlay using VXLAN + LISP + ISE for policy-based segmentation without VLANs everywhere
  • Assurance — collects telemetry from every device and uses AI/ML to detect issues before users notice them
  • Wireless Management — manages Catalyst 9800 WLCs and all APs; replaces the old Cisco Prime Infrastructure for wireless
  • REST API — full programmable interface for integration with ticketing systems, ITSM, and custom scripts

☁️

Cisco Meraki

Acquired by Cisco in 2012 — cloud-managed networking

Meraki is a cloud-managed networking platform — every Meraki device (switch, AP, firewall, camera) is managed 100% from the Meraki Dashboard at dashboard.meraki.com. There is no on-premises controller. There is no CLI by default. Configuration, monitoring, and firmware updates all happen through a web browser.

Meraki hardware phones home to Meraki's cloud infrastructure the moment it's powered on. You claim the device in the dashboard, apply a config template, and it self-configures. This makes Meraki extremely popular for distributed retail, branch offices, and SMB deployments where there's no on-site IT team.

⚠️

Licence dependency: Meraki requires an active annual licence per device. If the licence expires, the device still passes traffic but you lose dashboard access and the device eventually stops working. This is a critical commercial point in customer conversations.

Key Meraki concepts:

  • Dashboard — single browser UI managing all sites, devices, and users worldwide
  • Organisation > Network > Device — the three-level hierarchy in the dashboard
  • Config Templates — apply the same network config to 10 or 10,000 branches at once
  • Auto VPN — one checkbox creates hub-and-spoke IPSec tunnels between all MX firewalls automatically
  • No CLI required — everything is point-and-click, though a read-only CLI is available via SSH for troubleshooting

Catalyst Center vs Meraki — When to Use Which

Factor Catalyst Center (DNAC) Meraki
Management location On-premises appliance (data center) Cisco cloud (dashboard.meraki.com)
Hardware Existing Catalyst / IOS-XE devices Meraki-specific hardware only
CLI access Full IOS-XE CLI on all devices Minimal — dashboard-first design
Deployment complexity High — requires planning and expertise Low — zero-touch, ships pre-configured
Best for Large enterprise campus, data center Branches, retail, SMB, remote sites
Automation Full REST API, NETCONF, Ansible Meraki Dashboard API
Licensing model Perpetual + optional support contract Mandatory annual per-device licence
Internet dependency No — works fully on-premises Yes — devices need cloud connectivity to be managed
