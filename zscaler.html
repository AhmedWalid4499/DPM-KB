What is Zscaler?

Zscaler is the world's largest cloud security platform. Instead of routing all your internet traffic through a corporate firewall in a data center, Zscaler inspects it in the cloud — wherever the user is. It's one of the key services Orange Business offers to enterprise customers.

The Big Idea — In Plain English

❌ The Old Way — Hub & Spoke

In traditional networks, every branch office sent all internet traffic back through the corporate data center over an MPLS WAN link — even a simple Google search. The data center had the big firewall, so all traffic had to pass through it. This was fine when everyone sat in the office. But it's slow, expensive, and broken for remote workers and cloud apps.

Problems:
  • ✗ Remote users bypass the corporate network entirely
  • ✗ Cloud apps (Office 365, Salesforce) feel slow — traffic hairpins through DC
  • ✗ Expensive MPLS links become bottlenecks
  • ✗ One firewall = single point of failure

✅ The Zscaler Way — Security in the Cloud

With Zscaler, users connect to the nearest Zscaler data center (called a Point of Presence or PoP) — like connecting to a WiFi hotspot on the way to the internet. Zscaler inspects the traffic right there, applies all the security policies, and lets legitimate traffic through. No more hairpinning. No more VPN for basic browsing.

Benefits:
  • ✓ Same security policies for office, home, and travel users
  • ✓ Cloud apps connect directly — no backhauling
  • ✓ 150+ PoPs worldwide — users always connect to the nearest one
  • ✓ No hardware to maintain — fully managed cloud service
☁️

Think of Zscaler as a security checkpoint in the cloud. Every user — whether in the office, at home, or in a café — passes through this checkpoint before reaching the internet or internal apps. The checkpoint travels with the user, not the other way around.

How It Works — Step by Step

Here's exactly what happens when a user opens their browser and types a URL.

Every request passes through the Zscaler PoP — allowed or blocked before reaching the internet

1

User opens a browser or app

The Zscaler Client Connector (a lightweight app installed on the device) automatically intercepts all internet-bound traffic. The user doesn't have to do anything — no manual VPN connection required.

2

Traffic tunnels to the nearest Zscaler PoP

The Client Connector creates an encrypted TLS tunnel to the nearest Zscaler Point of Presence. With 150+ PoPs globally, latency is minimal — the user connects to a PoP in their city or country.

3

Zscaler inspects the traffic

Inside the PoP, Zscaler decrypts the TLS traffic (like an NGFW doing SSL inspection), checks the URL category, identifies the application (App-ID), scans for malware, checks DLP rules, and enforces the customer's security policy — all in milliseconds.

4

Allowed traffic goes directly to the internet / cloud app

If the request is permitted, Zscaler forwards it directly to its destination — Office 365, Salesforce, a website — from the PoP. No backhaul to a corporate data center. The user gets the fastest possible path to the cloud.

5

Blocked traffic is stopped and the user sees a block page

If the site is malicious, violates policy (e.g. gambling, malware C2 server), or contains sensitive data in a download, Zscaler blocks it and shows the user a customisable block page explaining why.

The Four Zscaler Products You Need to Know

Zscaler bundles its services under the Zero Trust Exchange platform. Here are the four you'll encounter most often at Orange Business.

🌐

ZIA — Zscaler Internet Access

Secures users accessing the public internet. This is the core product — it replaces the traditional perimeter firewall + web proxy. Includes URL filtering, SSL inspection, cloud sandboxing (malware analysis), DNS security, bandwidth control, and DLP. If a customer says "Zscaler", they almost certainly mean ZIA.

  • Full HTTPS/SSL traffic inspection at cloud scale
  • URL & category filtering (90+ categories)
  • Cloud Sandbox — runs unknown files to detect zero-day malware
  • DNS Security — blocks malicious domains at resolution
  • Bandwidth Control — throttle YouTube, limit Dropbox uploads
🔒

ZPA — Zscaler Private Access

Replaces the traditional VPN for accessing internal corporate applications. Instead of giving users full network access (as a VPN does), ZPA grants access only to the specific app they need — never exposing the rest of the network. This is the Zero Trust approach to remote access.

  • User connects to specific app, not the whole network
  • App Connector sits next to the internal app — no firewall hole needed
  • App is never exposed to the internet directly
  • Works for on-prem apps AND apps in AWS / Azure / GCP
  • Identity-aware: integrates with Azure AD, Okta, SAML
📊

ZDX — Zscaler Digital Experience

Monitors the end-to-end performance of every application — from the user's device, through the Zscaler PoP, all the way to the application server. When a user says "Teams is slow", ZDX can tell you exactly where the problem is: the user's WiFi, the ISP, the PoP, or Microsoft's servers.

  • Real-time latency and packet loss per user, per app
  • Device health monitoring (CPU, RAM, WiFi signal)
  • Path analysis — shows every hop between user and app
  • Pre-built dashboards for Teams, Zoom, Salesforce, O365
  • Alerts when user experience degrades below threshold
📱

ZCC — Zscaler Client Connector

The lightweight app installed on every user's device (Windows, Mac, iOS, Android) that silently redirects all traffic to Zscaler. Users don't see it or interact with it in normal operation. IT teams deploy it via MDM (Intune, JAMF) or GPO. Without ZCC, ZIA and ZPA don't work on the device.

  • Deployed automatically via Intune / JAMF / GPO
  • Runs silently — user doesn't need to "connect"
  • Chooses the best PoP automatically based on latency
  • Supports split tunnelling policies (bypass Zscaler for certain apps)
  • Sends device health data to ZDX for monitoring

Zero Trust — The Philosophy Behind Zscaler

🏰

Old model: "Castle and Moat"

The traditional network assumed that everything inside the corporate network is trustworthy. The firewall was the castle wall — once you got inside (via VPN or physically being in the office), you could reach almost anything. This fails catastrophically when an attacker gets inside — they can move freely across the entire network.

🔍

Zero Trust: "Never trust, always verify"

Zero Trust assumes that no user, device, or network should be trusted by default — even if they're inside the corporate office. Every access request is verified: Who is the user? What device? What app do they need? What time is it? What's their location? Only then is access granted — and only to the specific resource needed, nothing else.

💡

Why this matters for Orange Business: Many enterprise customers are moving from traditional MPLS + firewall setups to a Zscaler Zero Trust architecture as part of SD-WAN and SASE projects. When you're in a customer meeting and they mention "SASE", "Zero Trust", or "replacing our VPN" — Zscaler ZIA + ZPA is almost always the answer being discussed.

Orange Business + Zscaler — Common Customer Scenarios

🏢 Enterprise replacing MPLS with SD-WAN + Zscaler

Customer has 50 branches all backhauling internet traffic via MPLS to a central DC firewall. Orange Business replaces MPLS with broadband + SD-WAN (Cisco Viptela or Fortinet), and adds Zscaler ZIA at each branch. Traffic goes directly to the internet via Zscaler — MPLS cost drops by 60–80%.

🏠 Remote workforce securing home users

After COVID, 30% of employees work from home permanently. Old VPN is slow and overloaded. Orange Business deploys Zscaler ZCC on all laptops — home users get the same security policies as office users, with better performance since they hit a local Zscaler PoP instead of a VPN concentrator in a DC.

☁️ Moving internal apps to Azure / AWS

Customer is migrating on-prem SAP and internal tools to Azure. Old VPN can't easily secure access to cloud-hosted apps. ZPA App Connectors are deployed in the Azure VNet — users connect via ZPA without any public internet exposure of the app, from anywhere in the world.

🛡️ Regulatory compliance (GDPR, ISO 27001)

Customer needs to prove that all employee internet traffic is logged and inspected for compliance. Zscaler ZIA provides full SSL inspection logs, user-attributed activity reports, and DLP policies that block sensitive data (credit card numbers, PII) from being uploaded to personal cloud storage.

Key Zscaler Terms & Concepts

PoP (Point of Presence)
A Zscaler data center location where traffic is inspected. Zscaler has 150+ PoPs globally. Users are automatically routed to the closest one for lowest latency.
ZIA (Zscaler Internet Access)
The cloud-based secure web gateway — inspects all internet traffic. Replaces on-premises proxy, URL filter, and web firewall.
ZPA (Zscaler Private Access)
Zero Trust remote access to internal apps — replaces traditional VPN. Users access only the apps they're authorised for, never the whole network.
App Connector
A lightweight VM deployed next to an internal application. It creates an outbound tunnel to Zscaler — no inbound firewall hole needed. Users connect through ZPA → App Connector → App.
ZCC (Zscaler Client Connector)
The client app installed on user devices. Silently redirects traffic to Zscaler. Replaces the old VPN client. Deployed via Intune, JAMF, or GPO.
Tenant
A Zscaler tenant is one customer's Zscaler environment. Policies, users, and logs are isolated per tenant. Orange Business manages customer tenants on their behalf.
GRE / IPSec Tunnel
How branch office traffic reaches Zscaler when ZCC is not used. The SD-WAN router or firewall creates a GRE or IPSec tunnel to the nearest Zscaler PoP — all internet traffic is forwarded into the tunnel.
Zero Trust Exchange
Zscaler's name for their unified platform — ZIA + ZPA + ZDX + ZCC together. "Connecting users to apps, not to networks."
DLP (Data Loss Prevention)
Zscaler can scan outbound traffic for sensitive data (credit card numbers, PII, source code) and block uploads to unauthorised destinations like personal Dropbox or Gmail.
CASB (Cloud Access Security Broker)
Controls and monitors how employees use SaaS apps (Office 365, Slack, Box, Google Workspace). Zscaler CASB can block uploads of sensitive files to personal cloud storage while allowing corporate cloud storage.
