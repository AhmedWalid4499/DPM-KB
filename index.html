What does a Delivery Project Manager do?

A Delivery Project Manager (DPM) owns a customer's network delivery from the moment an order is placed through to final handover. You are the single point of accountability that turns a signed order into working infrastructure on the customer's site — keeping the project on schedule, on scope, and clearly communicated the whole way through.

🗂️

Own the delivery

Drive each project from kickoff to HOTO sign-off — planning the timeline, tracking milestones, and managing order records in tools like GOLD, ServiceNow, and SALTO.

🤝

Coordinate the teams

Orchestrate everyone the delivery depends on — TIMs and engineering, Field Engineers booked via FLIP, logistics, and the customer's own technical contacts.

📊

Manage scope & risk

Keep internal stakeholders and the customer aligned: confirm prerequisites, flag blockers early, run staging and dry runs, and document decisions as you go.

🔧

Run the migration

Execute LAN and WAN migrations — switches, WLCs and access points — manage UAT with the customer, then close out every system once the site is live.

The delivery lifecycle

Every DPM project follows the same three phases. The LAN Process guide breaks each one down step by step.

Phase 1

Pre-migration

Confirm supply, raise MACHX, validate prerequisites, stage hardware, and run dry runs with the customer.

Phase 2

Migration

Execute the cutover on the agreed window, support the field/TIM teams, and run UAT on every migrated service.

Phase 3

Post-migration / HOTO

Send the success notification, update DNAC/CMDB, upload the HOTO pack, and close the GOLD and SALTO orders.
