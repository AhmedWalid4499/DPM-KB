🔌 Layer 2

Switching, VLANs & Stacking

How switches forward traffic, how VLANs segment networks, how STP prevents loops, and how to stack multiple switches into one logical unit.

How Switches Work

📋

MAC Address Table

When a frame arrives, the switch records the source MAC address and the port it came from. Next time a frame is destined for that MAC, the switch sends it only to that port — not all ports. This is called unicast forwarding.

📢

Broadcast Domains

All ports on a switch are in the same broadcast domain by default — a broadcast frame goes to every port. VLANs split the switch into multiple smaller broadcast domains. A router (or Layer 3 switch) is needed to forward traffic between VLANs.

⚡

Cut-through vs Store-and-Forward

Store-and-forward: Switch receives entire frame, checks for errors, then forwards. Slower but reliable — used in most enterprise switches.
Cut-through: Forwards as soon as destination MAC is read. Faster but no error checking.

🔢

Layer 3 Switching

A Layer 3 switch (e.g. Cisco Catalyst 9300) can route traffic between VLANs at wire speed using SVIs (Switched Virtual Interfaces). Much faster than sending traffic to a separate router. Almost all enterprise core/distribution switches are L3.

VLANs — Virtual Local Area Networks

VLANs logically segment a single physical switch into multiple isolated networks. Traffic in VLAN 10 cannot reach VLAN 20 without going through a router or Layer 3 switch.

Three VLANs on one switch — isolated until L3 switch routes between them

Access Port

An access port belongs to a single VLAN. The connected device (PC, phone, printer) is unaware of VLANs — it just sees a normal network connection. Frames leaving the switch have no VLAN tag.

# Assign port to VLAN 10
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

Trunk Port

A trunk port carries multiple VLANs simultaneously using 802.1Q tagging. Used between switches, and between switches and routers/firewalls. Each frame carries a 4-byte VLAN tag with the VLAN ID.

# Configure trunk, allow VLANs 10, 20, 30
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
Switch(config-if)# switchport trunk native vlan 999
ℹ️

Native VLAN: On a trunk port, frames for the native VLAN are sent untagged. It's best practice to set the native VLAN to an unused VLAN (e.g. 999) to prevent VLAN hopping attacks.

Spanning Tree Protocol (STP)

STP prevents Layer 2 loops in networks with redundant switch paths. Without STP, a broadcast storm would bring down the entire network in seconds.

How STP Works

1

Root Bridge Election

The switch with the lowest Bridge ID (priority + MAC address) becomes the Root Bridge. Default priority is 32768.

2

Root Port Selection

Each non-root switch selects one Root Port — the port with the lowest cost path to the Root Bridge.

3

Blocked Ports

Redundant links are put into a "blocking" state. They don't forward traffic but can activate if a primary path fails.

STP Variants

ProtocolConvergenceNotes
STP (802.1D)30–50 secOriginal, rarely used now
RSTP (802.1w)<1 secRapid convergence, standard
PVST+<1 secCisco per-VLAN STP
MSTP (802.1s)<1 secMaps VLANs to instances
 
# Set this switch as Root for VLAN 10
Switch(config)# spanning-tree vlan 10 priority 4096
# Or use macro:
Switch(config)# spanning-tree vlan 10 root primary
⚡

PortFast + BPDU Guard: Enable PortFast on access ports connected to end devices — it skips the 30-second listening/learning delay. Add BPDU Guard to shut down the port if a switch is accidentally connected, preventing a rogue Root Bridge election.

Switch Stacking

Stacking connects multiple physical switches into one logical switch — managed as a single unit with a single IP address and single configuration.

🔼🔽 Uplinks vs Downlinks — What's the difference?

Before understanding stacking, you need to know these two terms — you'll hear them constantly in networking.

🔼 UPLINK

A port that connects upward toward the core of the network — toward a distribution switch, core switch, router, or firewall. Uplinks carry aggregated traffic from all the devices below. They are typically faster ports: 10G, 25G, or 100G SFP+ fibre connections, not regular copper 1G ports.

Example: a Catalyst 9300 access switch has 48× 1G PoE ports (downlinks) and 4× 10G SFP+ ports at the top — those 4 SFP+ ports are the uplinks.

🔽 DOWNLINK

A port that connects downward toward end devices — PCs, phones, printers, access points, and cameras. Downlinks deliver traffic to individual users or devices. They are typically 1G copper RJ45 ports, often with PoE to power devices over the same cable.

Example: those same 48 PoE ports on the 9300 are the downlinks — each one plugs into a PC, an AP, or an IP phone on a desk.

Cisco StackWise

Cisco's proprietary stacking technology. Switches connect in a ring using dedicated StackWise cables (not regular Ethernet). Stack bandwidth: 480 Gbps on StackWise-480 (Catalyst 9300). Up to 8 switches per stack. Master election based on highest priority or longest uptime.

  • Single management IP for entire stack
  • Seamless failover if master switch fails
  • Add/remove switches without downtime (hot-add)
  • Cross-stack EtherChannel for uplink redundancy

VSS / StackWise Virtual

For high-end Catalyst 9500/9600 switches, StackWise Virtual uses standard 10G/25G/100G interfaces as stack links — no proprietary cables needed. Two switches form a virtual switch pair. Used in distribution and core layers.

# Set stack member priority
Switch# switch 1 priority 15
Switch# show switch
# Renumber a stack member
Switch# switch 2 renumber 3

EtherChannel / LACP — Link Aggregation

EtherChannel bundles 2–8 physical links into one logical link for more bandwidth and redundancy. If one link fails, traffic automatically moves to the remaining links with no STP reconvergence. Uses LACP (IEEE 802.3ad, open standard) or PAgP (Cisco proprietary).

# Configure LACP EtherChannel on two ports
Switch(config)# interface range GigabitEthernet1/0/1 - 2
Switch(config-if-range)# channel-group 1 mode active   # LACP active
Switch(config-if-range)# switchport mode trunk
Switch(config)# interface port-channel 1
Switch(config-if)# switchport mode trunk

QoS — Quality of Service

QoS prioritizes important traffic (voice calls, video conferencing) over less time-sensitive traffic (file downloads) when a link is congested.

DSCP Marking

DSCP (Differentiated Services Code Point) is a 6-bit field in the IP header. Traffic is classified and marked close to the source (usually at the access switch), then queued based on its DSCP value.

DSCP NameValueUse Case
EF (Expedited Forwarding)46VoIP RTP media

BE (Best Effort)0General internet traffic

Queuing & Scheduling

When a port is congested, the scheduler decides which packets to send first. Common algorithm: LLQ (Low Latency Queuing) — gives voice traffic a strict priority queue, while other classes share remaining bandwidth fairly (CBWFQ).

💡

802.1p / CoS: On trunk ports, 802.1Q frames carry a 3-bit Class of Service value. Switches use CoS to prioritize frames at Layer 2, mapped to DSCP at the Layer 3 boundary.
