Key Differences Between the Two Processes

Both processes follow the same Pre / During / Post structure, but there are important differences a new DPM should know before starting.

Topic 📡 AP Process 🔌 Switch / WLC Process
Physical installation Customer mounts the APs themselves — DPM aligns this expectation in a pre-migration call Field Engineer is booked via FLIP — they handle physical hardware on-site
Staging TIMs stage the APs remotely before customer connects them Formal staging session is set up with customer and TIM; post-staging email is sent
Dry run Internal dry run with TIM and SC only External dry run with TIM, SC, and customer — 3rd party servers must be checked
Unique pre-check Option 43 must be verified — AP cannot find WLC without it Second MACHX raise required before migration (two total in pre-migration)
During migration UAT Customer tests all SSIDs first, then tests per-AP Customer tests all services connected to each migrated switch
Post-migration email Always send success notification Partial migration email if multi-day; success notification on final day
Tool: FLIP Not used Used to book the Field Engineer
MACHX raises One raise Two raises (pre-staging + pre-migration)
