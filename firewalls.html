🛡️ Security

Firewalls & Network Security

Next-generation firewalls, security zones, NAT, IPSec VPN, SSL inspection, IPS, and how Cisco, Palo Alto, and Fortinet approach network security.

Types of Firewalls

📋

Packet Filter (Stateless)

The simplest form — inspects each packet independently based on source/destination IP and port. No concept of "sessions". An ACL on a Cisco router is a stateless packet filter. Rarely used alone in modern networks.

🔄

Stateful Firewall

Tracks the state of network connections (TCP sessions, UDP flows). Knows which packets belong to an established, allowed session. Returns traffic is automatically permitted without an explicit rule. This is what Cisco ASA and most classic firewalls do.

🔬

NGFW — Next-Generation Firewall

Stateful + deep packet inspection at Layer 7. Can identify applications (e.g. YouTube, Teams, Dropbox) regardless of port. Includes IPS, URL filtering, malware sandboxing, and SSL/TLS decryption. All three vendors (Cisco FTD, Palo Alto, Fortinet) sell NGFWs.

☁️

Cloud / SASE Firewall

SASE (Secure Access Service Edge) delivers firewall-as-a-service from the cloud. Traffic from users and branches is tunneled to a cloud POP where policies are enforced. Examples: Palo Alto Prisma Access, Fortinet SASE, Cisco Umbrella.

Security Zones

Modern NGFWs group interfaces into zones. Security policy is written between zones — not between individual interfaces.

Three-zone model: untrust (internet), trust (LAN), DMZ (servers). Policy controls inter-zone traffic.

💡

DMZ (Demilitarized Zone): A buffer zone between untrust and trust. Place public-facing servers (web, mail, VPN gateway) here. Even if a DMZ server is compromised, the attacker still can't reach the internal LAN without crossing another firewall policy.

Security Policy / Rules

Rule Structure

A firewall rule matches on: Source zone + Source IP → Destination zone + Destination IP + Application/Port. If matched, the action is Permit, Deny, or Drop. Rules are evaluated top-to-bottom — first match wins.

Example policy (Palo Alto style):
Name: Allow-Corp-Web
From: trust → To: untrust
Source: 192.168.10.0/24
Dest: any
App: web-browsing, ssl
Action: allow + log

Implicit Deny

All firewalls have an implicit "deny all" at the bottom of the policy. If no rule matches, traffic is blocked. This is intentional — you must explicitly permit what you want to allow. Always log the deny rule so you can troubleshoot blocked traffic.

⚠️

Never put an allow any any rule in production. It defeats the purpose of a firewall. Always be as specific as possible with source, destination, and application.

VPN — Virtual Private Network

🔒

Site-to-Site IPSec VPN

Connects two branch offices securely over the internet. Both firewalls negotiate an IKE (Internet Key Exchange) phase to agree on encryption parameters, then establish an IPSec tunnel that encrypts all traffic between the sites.

  • IKEv2 is the modern standard (faster, more secure)
  • AES-256 + SHA-256 — recommended encryption
  • Pre-shared key or certificate-based auth
  • Dead Peer Detection (DPD) for tunnel health
👤

Remote Access VPN (SSL/IPSec)

Allows remote workers to securely connect to the corporate network. Client installs a VPN agent on their laptop.

  • Cisco AnyConnect / Secure Client — SSL + IPSec
  • Palo Alto GlobalProtect — SSL + IPSec
  • Fortinet FortiClient — SSL + IPSec
  • Split tunneling: only corporate traffic through VPN
  • MFA required — RADIUS/LDAP + TOTP

IPS & Threat Prevention

IPS — Intrusion Prevention System

IPS inspects traffic in real time against a database of known attack signatures. When a match is detected (e.g. a known exploit against Apache), the IPS can drop the packet, reset the connection, or alert. Signatures are updated by threat intelligence feeds (Cisco Talos, Palo Alto Unit 42, FortiGuard Labs).

SSL/TLS Inspection

~90% of internet traffic is now HTTPS. Without SSL inspection, the firewall can't see what's inside. The firewall acts as a man-in-the-middle: decrypts the traffic, inspects it, re-encrypts it. Requires deploying the firewall's CA certificate to client devices (via GPO/MDM).

URL Filtering

Blocks or monitors web traffic based on category (malware sites, adult content, social media, gambling). Fortinet uses FortiGuard categories; Palo Alto uses PAN-DB; Cisco uses Talos URL categories. Policy can log, block, or require authentication to override.

Sandboxing

Unknown files (email attachments, downloads) are sent to a sandbox environment to be executed and observed. If the file exhibits malicious behavior, it's blocked and a new signature is created. Palo Alto = WildFire; Fortinet = FortiSandbox; Cisco = Threat Grid.
